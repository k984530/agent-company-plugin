# 🤝 팀 회의 소집

팀 또는 전체 회사를 소집하여 협업 작업을 시작합니다.

## 소집 옵션

| 명령어 | 설명 |
|--------|------|
| `/meeting 조사팀 [작업]` | 조사팀만 소집 |
| `/meeting 토론팀 [작업]` | 토론팀만 소집 |
| `/meeting 마케팅팀 [작업]` | 마케팅제작팀만 소집 |
| `/meeting 조사검증팀 [작업]` | 조사검증팀만 소집 |
| `/meeting 토론검증팀 [작업]` | 토론검증팀만 소집 |
| `/meeting 마케팅검증팀 [작업]` | 마케팅검증팀만 소집 |
| `/meeting 전체 [작업]` | 6개 팀 순차 협업 (검증 게이트 포함) |

## 검증 게이트 프로세스

각 단계에서 검증팀이 작업을 검토하고, 다음 중 하나를 결정합니다:

### 판정 기준
| 결과 | 조건 | 다음 단계 |
|------|------|----------|
| ✅ 통과 | 검증 기준 충족 | 다음 팀으로 전달 |
| ⚠️ 수정 필요 | 사소한 문제 | 수정 후 통과 |
| ❌ 불통과 | 심각한 문제 | 재작업 요청 |

### 재작업 프로토콜
- **최대 재작업 횟수**: 3회
- **재작업 시**: 검증팀이 구체적인 수정 사항 전달
- **3회 불통과 시**: 에스컬레이션 (사용자에게 상황 보고)

## 실행 지침

1. **팀 파악**: 소집할 팀을 파악합니다
   - "전체" → 6개 팀 전체 소집 (검증 게이트 포함)
   - 특정 팀 이름 → 해당 팀만 소집

2. **에이전트 순차 실행**: `Task` 도구로 각 에이전트 실행

3. **검증 게이트 (전체 팀 협업 시)**:
   - 각 작업팀 완료 후 해당 검증팀 실행
   - 검증 결과에 따라 통과/재작업 결정
   - 재작업 횟수 추적 (최대 3회)

4. **결과 종합**: 최종 산출물 생성

## Task 도구 사용법

### 일반 에이전트 호출
```
Task:
  subagent_type: "agent-company:research-researcher"
  prompt: "[이전 결과 요약]\n\n다음 작업을 수행하세요: [작업 내용]"
  description: "researcher 작업 수행"
```

### 검증 에이전트 호출
```
Task:
  subagent_type: "agent-company:research-verify-qa-leader-kim-cheolmin"
  prompt: "[조사팀 결과물]\n\n위 결과물을 검증하고 통과/불통과를 판정하세요."
  description: "조사검증팀장 Kim Cheolmin 검증"
```

```
🔄 검증 게이트: [포함/미포함]
📊 예상 순서: [에이전트 순서]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

```
🔄 [에이전트 이름] 작업 중...
```

## 예시

### 전체 팀 협업
```
사용자: /meeting 전체 팀을 소집해서 "2025년 AI 트렌드 리포트" 작성해줘

→ 조사팀 작업
→ 조사검증팀 검증 (Kim Cheolmin, Lee Minjin, Park Seungho)
   └─ ✅ 통과
```
