# 🤝 팀 회의 소집

팀 또는 전체 회사를 소집하여 협업 작업을 시작합니다.

## 소집 옵션

| 명령어 | 설명 |
|--------|------|
| `/meeting 조사팀 [작업]` | 조사팀만 소집 |
| `/meeting 토론팀 [작업]` | 토론팀만 소집 |
| `/meeting 검증팀 [작업]` | 검증팀만 소집 |
| `/meeting 마케팅팀 [작업]` | 마케팅제작팀만 소집 |
| `/meeting 전체 [작업]` | 4개 팀 순차 협업 |

## 팀별 협업 흐름

### 🔍 조사팀 (Research)
```
researcher → data-collector → trend-analyst → benchmarker
    ↓              ↓               ↓              ↓
 정보 수집     데이터 추출      트렌드 분석    벤치마킹
```

### 💬 토론팀 (Debate)
```
facilitator → analyst → critic → synthesizer
     ↓           ↓         ↓          ↓
  논점 정리   장단점 분석   비판적 검토   결론 도출
```

### ✅ 검증팀 (Verification)
```
qa-leader → fact-checker → logic-validator
     ↓            ↓              ↓
 검증 계획    사실 확인      논리 검증
```

### 📢 마케팅제작팀 (Marketing)
```
content-director → copywriter → visual-designer → sns-manager
        ↓              ↓              ↓              ↓
   전략 수립       문안 작성      시각화 기획    SNS 최적화
```

## 전체 팀 협업 흐름

```
┌──────────────────────────────────────────────────────────┐
│                    📋 작업 요청                          │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│  🔍 조사팀: 정보 수집 및 분석                             │
│     researcher → data-collector → trend-analyst          │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│  💬 토론팀: 인사이트 토론 및 결론 도출                    │
│     facilitator → analyst → critic → synthesizer         │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│  ✅ 검증팀: 사실 확인 및 품질 보증                        │
│     qa-leader → fact-checker → logic-validator           │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│  📢 마케팅제작팀: 콘텐츠 제작                             │
│     content-director → copywriter → visual-designer      │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│                    ✅ 최종 산출물                         │
└──────────────────────────────────────────────────────────┘
```

## 실행 지침

1. **팀 파악**: 소집할 팀을 파악합니다
   - "전체" 또는 팀 이름이 없으면 → 4개 팀 전체 소집
   - 특정 팀 이름이 있으면 → 해당 팀만 소집

2. **에이전트 목록 확인**: `Glob`과 `Read`로 해당 팀의 에이전트 목록을 확인합니다

3. **작업 분배**: 각 에이전트에게 적절한 서브태스크를 할당합니다

4. **순차 실행**: `Task` 도구로 각 에이전트를 순차적으로 실행합니다
   ```
   Task(subagent_type: "agent-company:research/researcher", prompt: "...")
   ```

5. **결과 종합**: 모든 에이전트의 결과를 종합하여 최종 산출물을 생성합니다

## Task 도구 사용법

각 에이전트를 호출할 때 `Task` 도구를 사용합니다:

```
Task:
  subagent_type: "agent-company:{team}/{agent}"
  prompt: "[이전 에이전트 결과 요약]\n\n당신의 역할에 맞게 다음 작업을 수행하세요: [작업 내용]"
  description: "{agent} 작업 수행"
```
