# 🎉 새 에이전트 고용

사용자가 요청한 역할의 에이전트를 에이전트 회사에 추가합니다.

## 팀 구조

우리 회사는 4개의 팀으로 구성되어 있습니다:

| 팀 ID | 팀 이름 | 역할 |
|-------|---------|------|
| `research` | 🔍 조사팀 | 정보 수집, 데이터 분석, 트렌드 파악 |
| `debate` | 💬 토론팀 | 토론 진행, 장단점 분석, 의견 종합 |
| `verification` | ✅ 검증팀 | 사실 확인, 논리 검증, QA |
| `marketing` | 📢 마케팅제작팀 | 콘텐츠 제작, 카피라이팅, SNS 관리 |

## 고용 프로세스

1. **팀 확인**: 어느 팀에 배치할지 파악합니다
2. **역할 정의**: 새 에이전트의 역할과 책임을 정의합니다
3. **도구 선정**: 역할에 필요한 도구를 선정합니다
4. **파일 생성**: `agents/{team}/{name}.md` 형식으로 에이전트 파일을 생성합니다

## 사용 가능한 도구 목록

| 도구 | 용도 |
|------|------|
| `Read` | 파일 읽기 |
| `Write` | 파일 작성 |
| `Edit` | 파일 수정 |
| `Glob` | 파일 검색 |
| `Grep` | 내용 검색 |
| `WebSearch` | 웹 검색 |
| `WebFetch` | 웹 페이지 가져오기 |
| `Task` | 다른 에이전트 호출 |
| `mcp__hyperbrowser__*` | 브라우저 자동화 |
| `mcp__stitch__*` | UI 디자인 |

## 모델 선택 가이드

| 모델 | 용도 | 비용 |
|------|------|------|
| `haiku` | 단순 반복 작업, 데이터 수집 | 💰 저렴 |
| `sonnet` | 분석, 작성, 일반 업무 | 💰💰 보통 |
| `opus` | 복잡한 의사결정, 총괄 | 💰💰💰 비쌈 |

## 에이전트 파일 템플릿

새 에이전트 파일은 다음 형식을 따릅니다:

```markdown
---
name: {에이전트-영문-이름}
description: {한 줄 설명}
model: {haiku|sonnet|opus}
allowed-tools: {도구1}, {도구2}
---

# {에이전트 이름} ({이모지} {영문 이름})

## 역할
{역할 설명}

## 책임
- {책임 1}
- {책임 2}

## 작업 방식
1. {단계 1}
2. {단계 2}

## 출력 형식
{출력물 형식 설명}
```

## 실행 지침

1. 먼저 `Glob`으로 현재 에이전트 목록을 확인하세요
2. 사용자가 요청한 팀과 역할을 파악하세요
3. 적절한 영문 이름(kebab-case)을 생성하세요
4. 역할에 맞는 도구와 모델을 선택하세요
5. `Write` 도구로 에이전트 파일을 생성하세요
6. 생성 완료 후 환영 메시지를 출력하세요

## 예시

사용자: `/hire 조사팀에 번역가 추가해줘`

→ `agents/research/translator.md` 파일 생성
